Шаг 01

Контекст и ценность решения

Платформа закрывает разрыв между бумажными методиками и современным онлайн-сервисом. Психолог получает полный цикл: от сборки методики до выдачи отчётов.

3 роли админ · психолог · клиент
2 формата отчётов HTML и DOCX по запросу
0 регистрации для клиента вход по персональной ссылке

Шаг 02

Роли и правила доступа

Администратор

  • Создаёт аккаунты психологов, саморегистрация отключена.
  • Управляет сроком доступа и блокировкой вручную.
  • Видит аудит-лог критичных действий.

Психолог

  • Создаёт и клонирует методики в конструкторе.
  • Запускает кампании через именованные ссылки.
  • Получает результаты и формирует 2 вида отчётов.

Шаг 03

Сквозной сценарий платформы

  1. Психолог создаёт тест вручную или импортирует JSON-конфиг.
  2. Запускает кампанию и отправляет клиенту ссылку без регистрации.
  3. Клиент проходит анкету, прогресс и ответы сохраняются в БД.
  4. Система считает базовые и производные метрики по формулам.
  5. Готовые отчёты доступны сразу в HTML и DOCX.

Шаг 04

Killer-функции для демонстрации

Скорость сборки методики

  • Пресеты методик и формул в один клик.
  • Дублирование секций, вопросов и формул.
  • Автосохранение черновика с восстановлением.

Аналитика и презентация

  • Сравнение кампаний по источникам на одном экране.
  • Визуальные графики по среднему баллу и заполнению.
  • Экспорт результатов в CSV для внешней аналитики.

Шаг 05

Надёжность и прод-готовность

  • Rate limit на вход и отправку клиентской анкеты.
  • CSRF-защита всех POST-форм.
  • Автодеплой на VPS через GitHub Actions + smoke-check после релиза.
  • SMTP-уведомления в ключевых сценариях (welcome и отчёт клиенту).
