# Roadmap хакатона

## Фаза 0: Демо-готовность (уже сделано)
- Рабочий MVP со сквозным сценарием.
- Docker Compose и README для запуска.
- Seed-тест и демо-аккаунты.

## Фаза 1: Усиление UX (в ближайшие часы)
1. Упростить builder UX (лучшие подсказки, валидации, шаблоны вопросов).
2. Добавить toast-уведомления и обработку ошибок без white screen.
3. Подтянуть визуальное качество таблиц/карточек под презентацию.

## Фаза 2: Очки “звёздочка”
1. Клонирование теста.
2. Расширенный JSON import/export.
3. Графики метрик в HTML-отчётах.
4. Условная логика (минимум: скрытие/показ по yes/no).

## Фаза 3: Архитектурный апгрейд
1. Выделить frontend на Next.js.
2. Ввести Alembic-миграции.
3. Покрыть e2e/интеграционными тестами.

## Стратегия защиты
1. Сначала показать ценность для пользователя:
   - психолог создаёт методику за минуты
   - клиент проходит без регистрации
   - отчёты выходят сразу
2. Затем показать инженерную зрелость:
   - роли, доступы, архитектура, docker-compose
3. Завершить дорожной картой роста:
   - почему это база реального продукта, а не одноразовое демо
