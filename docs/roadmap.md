# Дорожная карта проекта

## Фаза 0: Демонстрационная база (уже реализовано)
- Рабочая минимальная версия со сквозным сценарием.
- Админская роль, роль психолога, клиентский проход без регистрации.
- Конструктор тестов + импорт/экспорт JSON.
- Генерация HTML/DOCX-отчетов по запросу.
- Деплой на VPS и автодеплой через GitHub Actions.

## Фаза 1: Закрыть P0 до 100%
1. Реализовать динамические дополнительные поля клиента по тесту.
2. Довести UX этого блока в конструкторе (шаблоны и валидация).
3. Проверить сценарий на демо-данных и зафиксировать в `docs/PROGRESS.md`.

## Фаза 2: Усиление качества (P1)
1. Миграции Alembic и документация по миграциям.
2. Централизованные страницы ошибок (400/403/404/500).
3. Логирование и диагностика старта приложения.
4. Интеграционные и модульные тесты.
5. CI: тесты + линтер при каждом push.

## Фаза 3: Дополнительные баллы (P2)
1. Drag-and-drop конструктор.
2. Ветвление сценариев в тестах.
3. Клонирование тестов и расширенный импорт/экспорт.
4. Визуальная аналитика и дашборд метрик.
5. Усиление разграничения прав для мульти-психологического режима.

## План защиты (структура презентации)
1. Показать ценность для пользователей:
   - психолог быстро собирает методику,
   - клиент проходит по ссылке без регистрации,
   - отчеты формируются сразу.
2. Показать инженерную зрелость:
   - архитектура, роли, деплой, автодеплой.
3. Показать рост продукта:
   - что уже сделано сверх минимума,
   - что запланировано в P1/P2.
