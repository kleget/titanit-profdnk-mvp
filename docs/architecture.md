# Архитектура минимальной версии

## Контуры системы
- Веб-интерфейс и API работают в одном сервисе на FastAPI.
- Данные хранятся в PostgreSQL через SQLAlchemy ORM.
- HTML и DOCX-отчеты генерируются в памяти по запросу (без сохранения файлов).

## Модули
- `routers/auth.py`: вход и выход.
- `routers/admin.py`: управление психологами и их доступом.
- `routers/psychologist.py`: кабинет психолога, тесты, результаты, отчеты.
- `routers/public.py`: публичная визитка и клиентское прохождение теста.
- `routers/api.py`: технические API-эндпоинты (`health` и т.д.).
- `services/tests.py`: создание тестов из UI/JSON и экспорт конфигурации.
- `services/scoring.py`: расчет базовых и производных метрик.
- `services/formulas.py`: безопасный вычислитель формул (песочница AST).
- `services/reports.py`: генерация HTML/DOCX-отчетов.
- `services/seed.py`: начальные данные для демо.
- `services/access_reminders.py`: напоминания об окончании доступа (7/3/1 день).

## Основные сущности
- `users`: админы и психологи, данные профиля, доступ, блокировка.
- `tests`: методики, настройки клиента, основной токен.
- `invite_links`: именованные ссылки кампаний для одного теста.
- `test_sections`: секции теста.
- `questions`: вопросы и параметры типов.
- `submissions`: прохождения клиентов и метрики.
- `answers`: ответы клиента по каждому вопросу.
- `metric_formulas`: пользовательские формулы производных метрик.

## Поток данных
1. Психолог создает тест вручную или импортирует JSON.
2. Для теста используется основной токен и при необходимости создаются именованные ссылки кампаний.
3. Клиент проходит тест по `/t/{token}` без регистрации.
4. Сервер валидирует поля, сохраняет ответы и считает метрики.
5. Психолог в кабинете видит результаты, группировки по меткам ссылок и генерирует отчеты.
6. Отчеты отдаются в HTML/DOCX на лету.

## Безопасность минимальной версии
- Сессионная авторизация через `SessionMiddleware`.
- Разделение прав по ролям на сервере.
- Проверка доступа к чужим тестам/результатам.
- Ограничение форматов загружаемых файлов (фото профиля).

## Ограничения текущей минимальной версии
- Пока нет Alembic-миграций (схема создается при старте).
- Нет drag-and-drop в конструкторе.
- Нет ветвления сценариев между вопросами.
- Нет полноценного CI с линтингом и автотестами.
