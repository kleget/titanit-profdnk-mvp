# Frontend V2 — Техническое задание и правила

## Цель
Полностью обновить интерфейс проекта в светлом минималистичном стиле без изменения backend-контрактов.

## Принципы UI
- Только светлая тема.
- Один шрифт: `IBM Plex Sans`.
- База: светло-серые нейтрали + мятный акцент.
- Сетка: кратность 8px.
- Радиусы: 10–12px (основные), 14–18px (крупные панели).
- Микро-анимации только для hover/focus/toast, без декоративных reveal-эффектов.

## Архитектура фронтенда
- SSR-шаблоны Jinja остаются основой.
- Глобальный каркас: `base.html` + единая шапка "верх + вкладки".
- Глобальные стили: `backend/app/static/css/app.css`.
- JS разделён по зонам:
  - `core-ui.js` — toast/уведомления и базовые UI-утилиты.
  - `builder.js` — логика конструктора (существующая бизнес-логика).
  - `builder-wizard.js` — пошаговая навигация конструктора.

## Карта экранов V2
- `login.html`: чистый вход без перегруза.
- `admin.html`: основной рабочий контур + сворачиваемые блоки "напоминания" и "аудит".
- `dashboard.html`: профиль, ключевые метрики, визитка и быстрые переходы.
- `tests.html`: список методик и быстрые действия (ссылка/клон/экспорт).
- `test_builder.html`: мастер из 5 шагов.
- `test_detail.html`: основной контур сверху, второстепенные блоки в сворачиваемых секциях.
- `client_test.html`: мобильный-first сценарий прохождения.
- `features.html`: лаконичный one-page для презентации.
- `business_card.html`, `client_done.html`, `error.html`: единый визуальный стиль.
- `reports/*.html`: print-first светлые отчёты.

## Совместимость и ограничения
- Не менять роуты, HTTP-методы и серверную логику.
- Не менять имена полей форм и CSRF-механику.
- Не менять URL-структуру отчетов и экспортов.

## Чеклист приемки
1. `pytest -q` проходит полностью.
2. На мобильном нет горизонтального скролла на рабочих экранах.
3. Основные действия доступны с первого экрана без лишних блоков.
4. Конструктор читается по шагам и не теряет старый функционал.
5. Отчеты (HTML/DOCX) продолжают генерироваться корректно.

## Правила атомарных коммитов
- Коммит 1: дизайн-система и базовый каркас (`base`, `app.css`, `core-ui`).
- Коммит 2: ключевые экраны роли (admin/psychologist/client).
- Коммит 3: отчеты + документация.
- Каждый коммит должен быть запускаемым и проходить тесты.
