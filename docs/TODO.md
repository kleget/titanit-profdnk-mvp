## Как работать с этим файлом
- Отмечаем `[x]` только после ручной проверки сценария в интерфейсе.
- Если задача сделана частично, оставляем `[ ]` и пишем короткую заметку в конце файла.
- Работаем сверху вниз: сначала `P0`, потом `P1`, потом `P2`.
- После закрытия заметной задачи добавляем запись в `docs/PROGRESS.md`.

## 8) Заметки
- Главный риск на текущем этапе: UX/UI воспринимается как «технический прототип», а не «продукт для пользователей».
- Главная точка роста по функционалу: лимиты ссылок + ветвления + сравнительная аналитика кампаний.
- Главная точка роста по защите: короткий, четкий, воспроизводимый демо-сценарий с быстрым сбросом данных.
