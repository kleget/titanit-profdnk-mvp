# UX/UI гайд проекта «ПрофДНК»

Последнее обновление: 2026-03-21

## Цель
- Единые правила интерфейса для всех экранов психолога, администратора и клиента.
- Минимум точечных визуальных исключений при доработке новых фич.
- Быстрая проверка качества перед демо и перед релизом.

## Сетка и отступы (8pt)
- Базовый шаг: `8px`.
- Разрешённые значения отступов/зазоров: `4, 8, 16, 24, 32, 40` (через CSS-токены `--space-*`).
- Основные контейнеры:
  - внешние отступы страницы: `24px` (desktop), `12px` (mobile).
  - внутренний отступ панели: `24px` (desktop), `16px` (mobile).
  - расстояние между секциями: `20-24px` в рамках одного экрана.
- Исключение по доступности:
  - интерактивная зона кнопок/ссылок в навигации может быть `44px` по высоте (требование mobile UX).

## Радиусы и карточки
- Радиусы:
  - малый: `10px` (`--radius-sm`)
  - средний: `14px` (`--radius-md`)
  - большой: `18px` (`--radius-lg`)
  - XL: `22px` (`--radius-xl`)
- Карточки/панели:
  - один стиль границы `1px solid var(--line)`.
  - один тип тени `--shadow-panel`.
  - не смешивать разные стили свечения в пределах одного экрана.

## Типографика
- Шрифты:
  - основной текст: `Manrope`.
  - бренд/акцентные заголовки: `Space Grotesk`.
- Иерархия:
  - `h1`: `clamp(28px, ...)`
  - `h2`: `clamp(24px, ...)`
  - `h3`: `clamp(20px, ...)`
  - body: `15-16px`
  - caption/hint: `13px`
- Для длинных заголовков и ячеек обязательно `overflow-wrap: anywhere`.

## Контролы форм
- Высота базового контрола: `48px` (`--control-height`).
- Кнопки:
  - основные и ghost-кнопки: минимум `44px`.
  - small-кнопки: тоже минимум `44px` для тач-доступности.
- Единые состояния:
  - `hover`: усиление границы/контраста.
  - `focus-visible`: явное кольцо фокуса.
  - `error`: класс `.is-invalid`.
  - `disabled`: сниженная непрозрачность и `cursor: not-allowed`.

## Mobile-first правила
- На мобильном:
  - sticky action bar в длинных формах.
  - sticky progress в клиентском тесте.
  - таблицы переходят в карточки (`responsive-cards-table`) без горизонтального скролла.
  - быстрые действия конструктора доступны в отдельной sticky-панели.
- Минимальная зона нажатия для интерактивов: `44px`.

## Пустые состояния (Empty State)
- Каждый пустой блок должен содержать:
  - краткое объяснение «почему пусто»;
  - конкретный следующий шаг;
  - по возможности кнопку действия (создать/добавить/скопировать ссылку).
- Формулировка без абстракций:
  - плохо: «Данных нет».
  - хорошо: «Пока нет прохождений. Скопируй ссылку и отправь клиенту».

## Чеклист перед демо
- Все страницы читаемы на ширинах `360`, `768`, `1280`.
- Нет горизонтального скролла на основных экранах.
- Все интерактивы доступны с клавиатуры (`Tab`, `Enter`, `Space`).
- Цветовой контраст в обеих темах достаточен для чтения.
- Пустые состояния содержат понятный next-step.
