# ПрофДНК: минимальная версия (ТИТАНИТ 2026)

Рабочая минимальная версия по кейсу хакатона: онлайн-платформа для профориентологов с конструктором методик, прохождением по ссылке без регистрации, сохранением результатов и генерацией двух типов отчетов (HTML + DOCX).

## Что уже реализовано

### Роли
- `admin`: создает психологов, управляет доступом и блокировкой.
- `psychologist`: создает тесты, работает с результатами и отчетами.
- `client`: проходит тест без регистрации по персональной ссылке.

### Сквозной сценарий
1. Психолог создает тест (через UI или импорт JSON).
2. Психолог отправляет клиенту ссылку.
3. Клиент проходит тест и отправляет ответы.
4. Система считает метрики и сохраняет прохождение.
5. Психолог получает клиентский и профессиональный отчеты.

### Покрытие кейса
- Конструктор с секциями и типами вопросов:
  - `text`, `textarea`, `single_choice`, `multiple_choice`, `yes_no`, `number`, `slider`, `datetime`, `rating`.
- Редактор формул производных метрик.
- Импорт/экспорт теста в JSON.
- Публичная визитка психолога + QR-код.
- Отчеты генерируются по запросу и не хранятся отдельными файлами на диске.
- Поддержка именованных ссылок кампаний и группировка результатов по источникам.

## Технологии
- Бэкенд: `Python 3.11`, `FastAPI`, `SQLAlchemy`.
- База данных: `PostgreSQL`.
- Интерфейс: `Jinja2 + vanilla JS`.
- Отчеты: `python-docx` + HTML-шаблоны.
- Развертывание: `Docker Compose`, `systemd`, `nginx`.

## Локальный запуск (без Docker)

1. Установить зависимости:
```bash
pip install -r backend/requirements.txt
```

2. Настроить переменные окружения:
- заполнить `.env` (или экспортировать переменные вручную);
- минимально нужны:
  - `APP_SECRET_KEY`
  - `DATABASE_URL`
  - `BASE_URL`
- рекомендовано:
  - `APP_ENV=production` для прод-сервера;
  - `SESSION_HTTPS_ONLY=true` на HTTPS-домене;
  - `SESSION_SAME_SITE=lax` (или `strict` при необходимости).
- `AUTO_SEED` по умолчанию выключен (`false`) и включается явно для демо-стендов.

3. Запустить приложение:
```bash
cd backend
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

## Автодеплой на VPS (GitHub Actions)
- Воркфлоу: `.github/workflows/deploy.yml`
- События запуска: `push` в `main` и ручной запуск `workflow_dispatch`.
- Целевой сервер: `recallio.tech`.
- На сервере выполняется:
  - обновление репозитория в `/srv/titanit-profdnk-mvp`,
  - обновление зависимостей Python,
  - перезапись `.env.deploy`,
  - `systemctl restart titanit-profdnk.service`,
  - `nginx reload`.

## Что добавили в последних итерациях

### Именованные ссылки и группировка по кампаниям
- У теста может быть несколько именованных ссылок.
- Прохождения группируются по метке источника.
- В деталях теста добавлена отдельная таблица групп и колонка источника.

Проверка:
1. Создать 2 именованные ссылки у теста.
2. Пройти тест по каждой ссылке.
3. Убедиться, что в результатах и группировке видны корректные метки.

### Динамические поля клиента + шаблоны полей
- В конструктор методик добавлен блок "Дополнительные поля клиента".
- Психолог может добавлять произвольные поля анкеты (тип, обязательность, плейсхолдер).
- Добавлены быстрые шаблоны наборов полей (например, "Школа/вуз", "Карьерная сессия").
- Поля рендерятся в клиентской форме, валидируются на сервере и попадают в отчеты.
- Для JSON-импорта поддержаны `custom_client_fields` и `client_fields.custom_fields`.

Проверка:
1. Открыть `/tests/new`, добавить 1-2 доп. поля и создать тест.
2. Пройти тест по публичной ссылке и заполнить эти поля.
3. Открыть клиентский и профессиональный HTML/DOCX-отчеты, убедиться что данные присутствуют.

### Адаптивная верстка и страница возможностей
- Улучшена мобильная верстка таблиц и длинных форм.
- Добавлена страница `/features` с обзором функций.
- Ссылка на возможности доступна из шапки и кабинета.

Проверка:
1. Открыть `/admin` и `/tests/{id}` на десктопе.
2. Проверить те же страницы в мобильной ширине (DevTools).
3. Открыть `/features` и убедиться, что страница читаема на всех экранах.

### Напоминания об окончании доступа (бонус)
- Добавлена логика напоминаний за `7 / 3 / 1` день.
- Админ видит список психологов с истекающим доступом.
- Психолог видит персональный баннер в кабинете.

Проверка:
1. В админке поставить психологу `access_until` на ближайшую дату.
2. Открыть `/admin` и увидеть блок напоминаний.
3. Войти под психологом и проверить баннер в `/dashboard`.

### Единая UI-система и улучшение мобильного UX (бонус)
- Введены единые токены отступов и радиусов для всех экранов.
- Обновлены формы и таблицы: одинаковые правила высоты контролов, фокуса, выравнивания и кнопок.
- Полностью переработаны интерактивные элементы теста:
  - styled checkbox/radio вместо стандартных браузерных,
  - удобный блок выбора `Да/Нет`,
  - рейтинг через кликабельные шкалы,
  - слайдер с текущим значением и диапазоном.
- Улучшен UX конструктора методик: более понятные названия типов вопросов и формулировки полей.

Проверка:
1. Открыть `/t/{token}` и пройти все типы вопросов, особенно `multiple_choice`, `yes_no`, `slider`, `rating`.
2. Проверить в мобильной ширине (DevTools), что элементы выбора удобны для тача и не ломают сетку.
3. Открыть `/tests/{id}`, `/tests/new`, `/admin` и проверить единые отступы/радиусы/выравнивание кнопок.

### Переключатель темы (приложение + HTML-отчеты)
- В шапке добавлен переключатель светлой/темной темы.
- Выбор сохраняется в `localStorage` и применяется на всех страницах приложения.
- В HTML-отчетах (`client` и `psychologist`) добавлен собственный переключатель с тем же сохранением.

Проверка:
1. В приложении нажать кнопку `Тема: ...` в шапке и перезагрузить страницу.
2. Открыть HTML-отчет и убедиться, что выбранная тема подтянулась.
3. Переключить тему прямо в отчете и проверить, что выбор сохранился.
